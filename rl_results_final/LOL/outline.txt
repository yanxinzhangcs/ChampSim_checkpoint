Introduction:
We have a decade of processor design which arguably has been about policies added to various parts of the out-of-order processor pipline like: cache replacement, prefetch, branch prediction. In general the philosophy is to build a processor with ONE selection of policies across these different things.

Question: do different workloads actually want different policies?
within a workload do different phases want different policies?
would a hypothetical (idealized) processor that had MULTIPLE policies perform much better than the typical static policy processor?

This work answers this question, as a way to explore whether it is fruitful to do research on dynamic policy selection as another know by which we can unlock single thread performance which is VERY hard to improve.

Methodology section:

- Champsim based.https://github.com/ChampSim/ChampSim
- list all the policies 
- we implemented them all in one code base
- we then chop up each workload into 100M instruction chunks (call that a timestep)
- we have 40 benchmarks, and 100 timesteps for each benchmark
